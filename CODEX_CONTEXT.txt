项目：astrbot_plugin_selfie_suite

作用：
- 单插件整合版自拍说说套件。
- 内置生活日程、自拍改图、QQ 空间发布三条链路。
- 目标是替代“life_scheduler + gitee_aiimg + qzone/selfie_bridge”多插件组合，同时尽量能与旧插件共存。

当前版本重点：
- v0.1.5
- 对外显示名改为“日常自拍发布助手”。
- 重点解决两个问题：
  1. 和旧三件套同时安装时的命令/注入冲突
  2. 自拍动作老一套、日程时段不够细的问题

模块结构：
- `main.py`
  - 主入口
  - 负责配置、命令、QZone 发布、自拍 prompt/caption 编排
- `internal_life/`
  - 日程数据、固定窗口、分段生成器
- `internal_aiimg/`
  - 参考图、改图路由、provider 注册、图片保存
- `internal_qzone/`
  - QQ 空间 session / api / model

本轮关键行为：
- 命令共存：
  - 源码装饰器和运行时注册表都统一到 `日常状态 / 刷新日常状态 / 日常刷新时间 / 自拍参考图 / 生成自拍 / 发布自拍说说`
  - 不再只依赖初始化后的运行时改名，减少和旧 life / aiimg / qzone / bridge 插件抢命令
- 注入共存：
  - 新增 `enable_llm_state_injection`
  - 新增 `suppress_injection_when_legacy_plugins_present`
  - 检测到旧插件存在时，默认关闭状态注入，避免重复往 system prompt 塞生活状态
- 自拍多样化：
  - 引入 `selfie_variant_state.json`
  - 对每个“日期 + 时段”稳定选择自拍动作、镜头、表情、场景和道具细节
  - 新增 `_SELFIE_VARIANT_COMMON_POOL`，把各时段专属池和全局补充池合并，避免动作老是那几套
  - 同一天同一时段保持稳定，跨天尽量去重
- 分时段细化：
  - life 生成器现在有本地时段变体池
  - 六个时段各自拥有独立的活动/地点/心情/自拍场景/自拍动作/光线/caption 提示
  - 新增 `_COMMON_SEGMENT_VARIATION_POOL`，让日程细节不只依赖单段的少量样本
  - 同一天生成稳定，不再完全依赖模型格式
- 文案兜底：
  - `_normalize_caption_text()` 会拦截 `I am ready to help...` 这一类英文废话，自动回退本地文案模板
- 用户可见输出：
  - `日常状态` 现在直接展示当前时段与全时段的穿搭细节、地点、心情、自拍动作和光线
- suite 单装兜底：
  - `main.py` 内置 `_default_life_config()`
  - 旧配置文件缺失 `pool` 时也能工作

调试路径：
- 冲突问题先看：
  - `main.py::_detect_legacy_conflict_plugins`
  - `main.py::_apply_command_namespace`
  - `main.py::_should_inject_llm_state`
- 自拍重复问题先看：
  - `main.py::_SELFIE_VARIANT_POOLS`
  - `main.py::_pick_selfie_variant`
  - `main.py::_build_segment_render_context`
- 日程过粗问题先看：
  - `internal_life/generator.py::_SEGMENT_VARIATION_POOL`
  - `internal_life/generator.py::_apply_local_segment_variation`
  - `internal_life/data.py::build_detailed_segments`
