# Changelog

## v0.1.8

- 插件设置页新增常用改图接口字段，不再只能手填 `embedded_aiimg_config_json`。
- 新增可视化字段：`aiimg_provider_type / aiimg_provider_id / aiimg_base_url / aiimg_api_url / aiimg_api_keys / aiimg_model / aiimg_timeout / aiimg_default_output / aiimg_proxy_url / aiimg_supports_edit`。
- 初始化时会把这些字段自动写入内部 aiimg 配置，并同步更新 `features.edit.chain` 与 `features.selfie.chain`，让自拍改图直接走设置页配置的接口。

## v0.1.7

- 新增 `publish_caption_enabled` 开关，可单独控制 `发布自拍说说` 时是否携带文案；关闭后仅发布图片。
- 重写默认 `caption_prompt_template`，更强调“像真人当天顺手发的短句”，避免总结感、广告感和提示词味道。
- 本地兜底文案改成按当前时段、活动、地点、心情、穿搭组合生成，更自然，不再总是同一种生硬句式。
- 发布成功回执和定时发布通知会根据开关状态展示“文案已关闭，本次仅发布图片”。

## v0.1.6

- 完善仓库对外信息，README 新增作者、仓库地址、安装地址和安装方式。
- GitHub 仓库侧将同步补充更完整的描述信息，方便直接从仓库页安装和识别用途。

## v0.1.5

- 插件显示名改为 `日常自拍发布助手`，不再使用“自拍说说一体化套件”这套偏生硬的名字。
- 整套命令重命名为更自然的说法：
  - `日常状态`
  - `刷新日常状态`
  - `日常刷新时间`
  - `自拍参考图`
  - `生成自拍`
  - `发布自拍说说`
- README、帮助文案和命令提示同步更新为新命名，减少用户安装后继续输旧命令的割裂感。

## v0.1.4

- 源码级命令入口改为 `套件日程 / 套件重写日程 / 套件日程时间 / 套件自拍参考 / 套件自拍 / 套件自拍说说`，不再只依赖初始化后的运行时改名，进一步缩短与旧三件套的冲突链。
- `查看日程` 输出改为直接展示当前时段与全部时段的穿搭细节、地点、心情、自拍动作和光线，用户可直接看出全天分段差异。
- 自拍改图默认提示词增加“禁止固定站桩、僵硬镜前摆拍、成熟职场写真、过熟妆造”的硬约束，强调青春少女感、真实生活感和贴近当前时段。
- 为自拍动作池和分时段日程池增加全局补充池，动作、镜头、场景、caption 提示不再只吃每段的少量固定样本。
- 文案归一化新增英文废话兜底拦截，像 `I am ready to help...` 这类无关输出会自动回退到本地文案模板。

## v0.1.3

- 运行时把套件命令统一改到 `套件*` 命名空间，减少和旧版 life / aiimg / qzone / bridge 插件的命令冲突。
- 新增 `enable_llm_state_injection` 与 `suppress_injection_when_legacy_plugins_present`，旧插件共存时默认抑制状态注入，避免双重注入。
- life 内核补齐默认配置池，单独安装时也能稳定生成分时段日程。
- 日程生成改成“同一天固定、分时段细化、局部多样化”的策略：同一天重复生成结果稳定，不同时间段的活动/地点/自拍场景更细。
- 自拍生成改为读取当前时段的穿搭、活动、地点、情绪、自拍提示，并加入动作/镜头/表情/道具变体缓存，减少动作老一套的问题。
- 重写 `_conf_schema.json` 为干净 UTF-8，并补充新的共存配置项。

## v0.1.2

- 修复 GitHub 仓库缺失内部目录文件，导致 AstrBot 安装后导入 `internal_aiimg` 失败的问题。
- 为 `internal_aiimg/core` 补充显式包入口，降低安装环境对命名空间包解析的兼容风险。
- 修正 `metadata.yaml` 的描述、版本号与仓库地址。

## v0.1.1

- 切为彻底独立运行版。
- 不再默认复用旧插件配置文件。
- 不再默认复用旧插件数据目录。
- 不再接管外部 qzone 插件发帖流程。
- 插件配置页移除旧配置复用相关选项。

## v0.1.0

- 新增单插件整合版 `astrbot_plugin_selfie_suite`。
- 内嵌生活日程、自拍改图和 QQ 空间发布底层链路。
- 覆盖自拍说说主链、QZone 登录预检、失败恢复、通知与定时发布。
